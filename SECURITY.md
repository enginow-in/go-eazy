# Security Policy

The security of **Go-Eazy** is our top priority. We take security vulnerabilities very seriously and appreciate the efforts of security researchers and the community in keeping our project safe.

## 🛡️ Supported Versions

We provide security updates for the following versions of the project. We strongly encourage all users to stay on the latest stable release.

| Version | Supported          | Notes                                      |
| ------- | ------------------ | ------------------------------------------ |
| 5.1.x   | :white_check_mark: | Active support and latest security patches |
| 4.0.x   | :white_check_mark: | Critical security patches only             |
| < 4.0   | :x:                | End of life; no longer supported           |

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability within Go-Eazy, please do not disclose it publicly. Instead, report it to us confidentially.

**How to Report:**
1. Send an email to **[contact@enginow.in](mailto:contact@enginow.in)** with the subject `[Security Vulnerability] - <Brief Description>`.
2. Include a detailed description of the vulnerability, including steps to reproduce it.
3. Provide any potential impact and, if possible, a proof of concept (PoC).

**What to Expect:**
- We will acknowledge receipt of your vulnerability report within **48 hours**.
- We will provide a preliminary assessment and timeline for a fix within **7 days**.
- You will receive regular updates on our progress until the issue is resolved.
- Once the vulnerability is patched, we will coordinate public disclosure and, if you wish, credit you for the discovery.

Thank you for helping us keep Go-Eazy secure! 🔒
